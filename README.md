# Угадай сколько шариков — деплой на Render

Простое приложение: форма на "/" для гостей и панель "/admin" для организатора.
Данные хранятся в PostgreSQL (бесплатная база от Render), поэтому не теряются
при "засыпании" бесплатного веб-сервиса.

## Шаг 1. Выложить код на GitHub

1. Создайте новый репозиторий на github.com (например, `guess-the-jar`).
2. Загрузите туда все файлы из этой папки: `server.js`, `package.json`, `README.md`.
   Проще всего — через кнопку "Add file → Upload files" на странице репозитория,
   без работы в терминале.

## Шаг 2. Создать бесплатную базу данных на Render

1. Зайдите на render.com, зарегистрируйтесь / войдите.
2. Нажмите **New +** → **PostgreSQL**.
3. Укажите любое имя (например, `guess-jar-db`), регион — любой, план — **Free**.
4. Нажмите **Create Database** и подождите, пока она поднимется.
5. Откройте созданную базу и скопируйте значение **Internal Database URL**
   (понадобится на следующем шаге).

## Шаг 3. Создать веб-сервис

1. На Render нажмите **New +** → **Web Service**.
2. Подключите ваш GitHub-репозиторий с этим кодом.
3. Настройки:
   - **Runtime**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Plan**: Free
4. В разделе **Environment Variables** добавьте:
   - `DATABASE_URL` = (вставьте Internal Database URL из шага 2)
   - `ADMIN_PASSWORD` = свой код доступа для `/admin` (например, `1979`)
5. Нажмите **Create Web Service**.

Render соберёт и запустит приложение. Через минуту-две вы получите постоянную
ссылку вида `https://guess-jar.onrender.com` — это и есть ваша форма.

## Шаг 4. Проверить

- Откройте `https://ваш-адрес.onrender.com/` — должна открыться форма.
- Отправьте тестовый ответ, попробуйте отправить второй раз с тем же именем —
  должно быть отклонено.
- Откройте `https://ваш-адрес.onrender.com/admin`, введите код доступа,
  проверьте список и сохранение правильного ответа.

## Важно про бесплатный тариф Render

- Бесплатный веб-сервис "засыпает" после ~15 минут без запросов и просыпается
  примерно за 30–50 секунд при первом заходе — это нормально, гость просто
  подождёт немного при открытии ссылки.
- Данные не теряются при "засыпании", потому что хранятся в отдельной базе
  PostgreSQL, а не на диске самого веб-сервиса.
- Бесплатная база Render активна первые 30 дней — этого достаточно для
  мероприятия на несколько дней. После этого её можно удалить.

## QR-код

Когда получите постоянную ссылку, сгенерируйте QR-код через любой бесплатный
сервис (например, qr-code-generator.com), просто вставив туда вашу ссылку.
